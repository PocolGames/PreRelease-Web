# PreRelease - 사용 가이드 📚

## 🎯 개요
PreRelease는 Twitter 스타일의 블랙 테마 SNS 홈페이지입니다. Markdown을 지원하며, 여러 장의 이미지와 함께 포스팅할 수 있습니다.

## 📁 폴더 구조
```
PR_Web/
├── index.html              # 메인 페이지
├── css/
│   ├── style.css          # 메인 스타일
│   └── responsive.css     # 반응형 디자인
├── js/
│   ├── main.js           # 메인 JavaScript
│   └── markdown.js       # Markdown 파서
├── images/               # 이미지 파일들
├── posts/
│   └── posts-data.js     # 포스팅 데이터
└── README.md             # 이 파일
```

## ✍️ 새 포스팅 추가하기

### 1단계: 이미지 준비 (선택사항)
- 사용할 이미지들을 `images/` 폴더에 복사합니다
- 지원 형식: JPG, PNG, GIF, WebP
- 권장 크기: 800x600 이상

### 2단계: 포스팅 데이터 추가
`posts/posts-data.js` 파일을 열고 `postsData` 배열에 새 객체를 추가합니다:

```javascript
{
    id: 6,                    // 고유 번호 (기존 번호보다 큰 수)
    date: "2025-09-10",       // 날짜 (YYYY-MM-DD 형식)
    content: `포스팅 내용`,    // Markdown 형식 지원
    images: [                 // 이미지 배열 (선택사항)
        "images/photo1.jpg",
        "images/photo2.jpg"
    ]
}
```

### 3단계: 저장 및 확인
- 파일을 저장합니다
- 브라우저에서 `index.html`을 새로고침합니다

## 🖼️ 이미지 기능

### 이미지 개수별 레이아웃
- **1장**: 전체 너비
- **2장**: 2x1 그리드
- **3장**: 좌측 큰 이미지 + 우측 2개 작은 이미지
- **4장**: 2x2 그리드
- **5장 이상**: 2x2 그리드 + "+숫자" 표시

### 이미지 클릭
- 이미지를 클릭하면 큰 모달창에서 볼 수 있습니다
- ESC 키 또는 배경 클릭으로 닫을 수 있습니다

## 🎨 커스터마이징

### 색상 변경
`css/style.css` 파일에서 다음 색상들을 수정할 수 있습니다:
- `#000000`: 배경색
- `#ffffff`: 텍스트 색
- `#1d9bf0`: 포인트 색상 (로고, 링크 등)
- `#71767b`: 보조 텍스트 색
- `#2f2f2f`: 경계선 색

### 폰트 변경
`css/style.css`의 `font-family` 속성을 수정하세요.

### 레이아웃 조정
- 최대 너비: `.container` 클래스의 `max-width` 수정
- 여백: 각 요소의 `padding`, `margin` 수정

## 🔧 고급 기능

### 날짜 형식
- 자동으로 "몇 일 전", "몇 주 전" 형식으로 표시됩니다
- 한 달 이상 지나면 전체 날짜가 표시됩니다

### 반응형 디자인
- 데스크톱, 태블릿, 모바일에서 모두 최적화되어 있습니다
- `css/responsive.css`에서 세부 조정이 가능합니다

### 성능 최적화
- 이미지 lazy loading을 추가하려면 `js/main.js`를 수정하세요
- 무한 스크롤 기능은 이미 준비되어 있습니다

## 🚀 배포하기

### GitHub Pages
1. GitHub 저장소에 업로드
2. Settings > Pages에서 배포 설정
3. `index.html`이 루트에 있는지 확인

### Netlify
1. 폴더를 zip으로 압축
2. Netlify에 드래그 앤 드롭
3. 자동으로 배포됩니다

### 일반 웹 호스팅
- 모든 파일을 웹 서버의 public 폴더에 업로드하세요

## 🆘 문제 해결

### 포스팅이 보이지 않을 때
1. `posts/posts-data.js`에서 문법 오류 확인
2. 브라우저 개발자 도구(F12)에서 오류 메시지 확인
3. 이미지 경로가 올바른지 확인

### 이미지가 로드되지 않을 때
1. 파일 경로와 이름 확인 (대소문자 구분)
2. 이미지 파일이 `images/` 폴더에 있는지 확인
3. 지원되는 이미지 형식인지 확인

### 스타일이 적용되지 않을 때
1. CSS 파일 경로 확인
2. 브라우저 캐시 삭제 후 새로고침
3. 개발자 도구에서 CSS 로딩 상태 확인
